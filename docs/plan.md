# Final Project – Midterm Plan

## Deploy Link
https://rendezvous-rho.vercel.app

## 1. 本次 Prototype 已完成
- 基本頁面架構（首頁餐廳 / 揪團貼文串、使用者檔案頁、收藏餐廳 / Collection 頁，透過 React Router 串起多頁導覽）
- 三欄式版面與導覽列（上方導覽列與搜尋列、左側進階篩選側欄、中間貼文串、右側地圖與「You might like」推薦側欄）
- 餐廳評價貼文系統 prototype（評價卡片含星等、價格區間、標籤、多張圖片輪播 / lightbox、按讚 / 留言 / 分享等互動 UI，以假資料示意）
- 「揪吃飯貼文串」示意功能（揪團貼文卡片顯示餐廳名稱與地點、用餐時間、預計人數與預算、訂位狀態、可見範圍與招募狀態，並有建立揪團的 composer 視窗 UI）
- 進階篩選與搜尋（左側側欄提供風格 / 類別 / 評價 / 價格 / 距離等多條件篩選，標籤點擊即可過濾，頂部搜尋列支援餐廳名稱與地點關鍵字搜尋）
- 使用者檔案頁 prototype（頭像、暱稱、帳號名稱、簡介、偏好標籤、追蹤 / 粉絲數與分頁切換，含基本個人檔案編輯視窗 UI）
- 收藏餐廳頁面與地圖初步實作（Map / List 雙模式切換、Google Maps 底圖與餐廳標記、地點搜尋、自訂餐廳卡片與細節面板）
- 登入框架（Google 登入入口與 UI 位置已預留，尚未串接實際後端與 OAuth 流程，目前為停用 / placeholder 狀態）
- 前端 ThemeContext 與設定架構雛形（已有主題相關 context 與樣式設計，尚未完成實際的 Settings 頁面與主題切換按鈕）

## 2. 最終版本預計完成項目
- 完整的使用者流程：Google 登入與登出、個人檔案建立與編輯、偏好設定（喜好菜系 / 價位 / 地區）、瀏覽餐廳評價貼文、發佈 / 編輯 / 刪除貼文、「揪吃飯貼文串」揪團與參加流程，以及收藏與管理餐廳 / 清單
- 資料庫串接：建立後端服務與線上資料庫（儲存使用者、餐廳、評價貼文、「揪吃飯貼文串」、收藏清單與清單項目），設計完整 schema 與 API，讓前端可進行實際的 CRUD 與狀態持久化
- 三大主要功能：「揪吃飯貼文串」（揪團發文、報名、成團 / 結束流程）、「餐廳評價與分享」（圖文評價、標籤與搜尋 / 篩選、社交互動）、「收藏清單與食物推薦器」（List Collection 頁面 + 類似 Google Maps 的收藏清單檢視、根據使用者偏好與行為的餐廳 / 清單推薦）
- 其他：Settings 頁面（通知與推薦偏好設定）、深淺色主題切換器（Theme Switcher）、更完善的 RWD / UI/UX 優化（loading 狀態、錯誤提示、表單驗證與互動細節）、追蹤 / 推薦系統優化（依追蹤對象與互動調整推薦內容）、repost 功能與專案 logo / 品牌設計

## 3. 預期開發進度
- Week 1: 完成 Google 登入與基本使用者模型（後端 OAuth 串接）、建立資料庫 schema（users / restaurants / posts / meetup_posts / collections 等），並實作取得餐廳與基本貼文列表的 API，讓現有前端頁面可改用真實資料來源
- Week 2: 完成「揪吃飯貼文串」與餐廳評價的 CRUD（前後端串接）、實作收藏餐廳與 Collection 的新增 / 刪除 / 編輯流程，將 List Collection 頁面與地圖整合，讓使用者可以在地圖與清單中管理收藏，同步完成專案 logo 與基本品牌視覺、repost 功能的初版流程設計
- Week 3: 完成食物推薦器（依據收藏紀錄、標籤與偏好產生推薦）、Settings 頁面與 Theme Switcher（含深淺色主題記憶）、針對追蹤 / 推薦系統做優化調整，並將整體網站調整為更貼近 app 的使用體驗（RWD 強化、PWA / app-like 安裝體驗考量），進行效能與穩定性修正，串起完整 demo 流程並整理最終文件與展示內容
